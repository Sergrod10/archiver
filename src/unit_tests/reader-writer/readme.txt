Тест имеет вид:
n

Потом n строк вида: 

x1 b1
x2 b2
...
xn bn

Где xi - символ, который нужно считать/вывести, а bi - кол-во битов, сколькими закодировать xi(очевидно, что должно выполняться xi < 2^bi)

Для удобства каждый файл я буду сначала выводить, а потом считывать то, что вывел. Таким образом я сразу же смогу проверить и writer и reader.

При запуске программы надо передавать 1 аргумент: название папки теста

Например: g++ -std=c++20 test.cpp writer.cpp reader.cpp && ./a.out test1
